Security Review Checklist:
- Dependencies: Checked using `cargo audit`. Two vulnerabilities found in `ring` and `rustls-webpki` via `reqwest`.
- Code Execution/Injection: Investigated `Command::new()`. Found safe usage with `args()` taking individual string parameters (`script_args`) — not `shell=True`. 
- Memory Safety: Searched for `unsafe {`. Zero occurrences in `crates/`.
- Panics/Unwraps: Checked for `.unwrap()`. The vast majority are in `cfg(test)` code. The few in production code (like `fs.rs:320`) are guarded by preceding `is_err()` checks, so they will not panic.
- Input validation: Looked for `unwrap_or_default()`. Commonly used in CLI argument parsing and memory block loading (safe).
- Telemetry/Network: The agent connects to `.env` server URLs (`CADE_SERVER_URL`), but only executes tools explicitly invoked by the model.
- Path protection: Verified `path_is_protected` blocks writes to `.env`, `.git/`, `.ssh/`, and `.cade-db`.
