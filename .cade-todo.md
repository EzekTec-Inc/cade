# CADE v2 IDE Integration Strategy: MCP Server Blueprint

## Architectural Paradigm
To keep `cade-server` bloat-free and editor-agnostic, IDE integration will be achieved by building an **IDE-specific MCP Server** inside the respective editor's extension/plugin ecosystem. CADE core will simply connect to this server as a standard MCP client.

## Target Implementations

### 1. Visual Studio Code Extension (`cade-vscode`)
- **Language:** TypeScript / Node.js
- **SDK:** `@modelcontextprotocol/sdk`

### 2. Neovim Plugin (`CADE-nvim`)
- **Language:** Python (via `pynvim` and `mcp` SDK) or Lua.
- **Transport:** Stdio (piped to CADE)
- **Mechanism:** The MCP server runs as a separate process attached to Neovim's RPC socket (`NVIM_LISTEN_ADDRESS`).
- **Resource Mappings:**
  - `ide://active-editor/buffer` -> Reads `nvim.current.buffer`

## Phase 1: Neovim Proof of Concept (Python RPC Bridge)

**Goal:** Create a standalone MCP server that reads the active Neovim buffer.

**Step 1: The Environment**
Create a new directory `CADE-nvim/mcp-server` alongside the CADE project.
Create a virtual environment and install the dependencies:
```bash
python3 -m venv venv
source venv/bin/activate
pip install mcp pynvim
```

**Step 2: The MCP Server Script (`server.py`)**
A minimal Python script that connects to Neovim and serves the MCP resource over stdio:
```python
import asyncio
import os
import pynvim
from mcp.server import Server
from mcp.server.stdio import stdio_server

# 1. Connect to the active Neovim instance via its socket
nvim_socket = os.environ.get("NVIM_LISTEN_ADDRESS")
if not nvim_socket:
    raise RuntimeError("NVIM_LISTEN_ADDRESS environment variable is required")

nvim = pynvim.attach('socket', path=nvim_socket)

# 2. Initialize the MCP Server
app = Server("cade-nvim-mcp")

# 3. Expose the active buffer as a resource
@app.read_resource("ide://active-editor/buffer")
async def read_buffer(uri: str) -> str:
    # Fetch the current active buffer contents from Neovim
    buf = nvim.current.buffer
    text = "\n".join(buf[:])
    return text

if __name__ == "__main__":
    # Start the MCP stdio server loop
    async def main():
        async with stdio_server() as (read_stream, write_stream):
            await app.run(read_stream, write_stream, app.create_initialization_options())
    asyncio.run(main())
```

**Step 3: Connecting CADE**
In CADE, you will configure the MCP connection (via `~/.cade/settings.json` or `/mcp`) to run this Python script, passing the Neovim socket:
```json
{
  "mcp_servers": {
    "cade-nvim": {
      "command": "/path/to/CADE-nvim/mcp-server/venv/bin/python",
      "args": ["/path/to/CADE-nvim/mcp-server/server.py"],
      "env": {
        "NVIM_LISTEN_ADDRESS": "/tmp/nvim.pipe"
      }
    }
  }
}
```

**Step 4: Testing**
Start Neovim with a named socket: `nvim --listen /tmp/nvim.pipe`
Open CADE, and ask: "Read the resource `ide://active-editor/buffer` and tell me what is in my editor." CADE will query the Python MCP server, which queries Neovim via RPC, and returns the live unsaved text.
