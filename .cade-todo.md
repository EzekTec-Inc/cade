# Investigation: /memory Menu Modernization

The user requested an investigation into modernizing the `/memory` screen (displayed when selecting `/memory` from the `/help` menu) to match the tabular, fuzzy-filtering layout we previously implemented for the `/agents` picker.

## 1. Current Implementation
When a user executes `/memory` with no arguments, the CLI fetches all memory blocks from `cade-server` and prints them linearly directly into the terminal's standard output stream (using `self.tui_hdr` and `self.tui_dim`). It does not use a full-screen interactive modal.

## 2. Proposed Modernization Plan
To bring this in line with our recent UI upgrades without adding bloat:
1. **Extract to `pickers.rs`**: Create a new `pub(crate) async fn memory_picker(...)` in `crates/cade-cli/src/cli/repl/pickers.rs`.
2. **Tabular Layout**: Replace the linear printout with a `ratatui::widgets::Table`. The columns should explicitly show `[Tier, Label, Length, Updated At, Description]`.
3. **Interactive Fuzzy Filtering**: Add the same split-pane layout from the `/agents` picker, placing a `Filter (Type to search)` text box at the bottom. As the user types, the list of memory blocks will dynamically filter by matching the query against the block's `label` or `description`.
4. **Right-Pane Preview (Optional but High Value)**: Since memory block values can be large (up to thousands of characters), we can split the top pane horizontally: 
   - Left side: Table of memory blocks.
   - Right side: A `Paragraph` rendering the full `value` of the currently highlighted memory block in real-time.
5. **Hotkeys**: Implement native hotkeys within the modal to manage memory:
   - `Enter` / `v`: View/Edit block
   - `d` / `Delete`: Delete block
   - `p`: Pin / Unpin block

## 3. Integration
In `crates/cade-cli/src/cli/repl/slash.rs` (or where `SlashCmd::Memory` is processed), if the subcommand is empty (`_`), we await the `memory_picker` instead of looping over and printing the blocks.

This approach will massively improve the discoverability and UX of the agent's memory system while strictly reusing the zero-bloat `ratatui` paradigms established in the previous refactor.
