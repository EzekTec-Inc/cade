## Implementation Plan: Secure `Plan` Mode

### 1. Update Settings Schema
- Modify `crates/cade-core/src/settings/manager.rs` to add a new flag inside `PermissionSettings`:
  ```rust
  #[serde(default)]
  pub allow_agent_mode_changes: bool,
  ```
- Update `GlobalSettings` and its default initialization so it defaults to `false` (meaning the LLM cannot elevate its own permissions).

### 2. Guard the Tool Schemas
- In `crates/cade-agent/src/tools/manager.rs`, the `all_schemas(toolset)` function currently injects `EnterPlanModeTool::schema()` and `ExitPlanModeTool::schema()` unconditionally.
- **Change:** Pass the `PermissionSettings` (or just the `allow_agent_mode_changes` boolean) down into `all_schemas()`. Only append the `EnterPlanMode` and `ExitPlanMode` schemas to the available toolset if this setting is `true`.
- **Result:** When `false`, the LLM physically will not know these tools exist, making it impossible for it to autonomously break out of `Plan` mode.

### 3. Guard the Intercept Layer (Defense in Depth)
- In `crates/cade-cli/src/cli/repl/turn_loop.rs` (`try_native_intercept`), locate the handlers for `"EnterPlanMode"` and `"ExitPlanMode"`.
- **Change:** Add a check against `self.settings.lock().unwrap().permission_settings().allow_agent_mode_changes`. If the LLM somehow hallucinates a call to these tools when they are disabled, intercept it and return an error `ToolResult`:
  ```json
  "output": "Permission denied: agent mode changes are disabled in settings.json"
  ```

### 4. Testing
- Update `tests/approval_tests.rs` (or similar permission tests) to verify that `ExitPlanMode` is correctly blocked and its schema hidden when `allow_agent_mode_changes` is `false`.
- Ensure the project builds and all tests pass cleanly under the `strict-project-execution` and `rust-developer` directives.
