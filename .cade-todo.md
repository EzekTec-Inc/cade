# Cross-Platform Desktop Refactor Plan
*Status: In Progress*

## Phase 1: Input & Window Control (`desktop/control.rs`)
*Status: Completed*
- Added `enigo = "0.6.1"` and `active-win-pos-rs = "0.10.0"`.
- Removed `xdotool` and `ydotool` subprocess execution.
- Implemented `DesktopControl` using native OS APIs via `enigo` for keyboard input and mouse movements. 
- Integrated `active-win-pos-rs` for window detection.

## Phase 2: System Tray (`desktop/tray.rs`)
*Status: Completed*
- Removed Linux-only `ksni` dependency from the workspace.
- Added `tray-icon = "0.21.3"`.
- Rewrote the tray initialization to build a `tray_icon::TrayIcon` with a native menu that works seamlessly across Windows, macOS, and Linux without blocking the main event loop.

## Phase 3: OS Notifications (`desktop/notify.rs`)
*Status: Investigated*
- Evaluated `notify-rust` feature sets. Confirmed it natively supports macOS via `mac-notification-sys` out of the box and Windows natively via D-Bus wrappers/WinRT integrations in default features. No feature-flag changes were strictly necessary as the library compiles correctly on all targets.

## Phase 4: Screen Capture (`desktop/capture.rs`)
*Status: Completed (No-op)*
- `xcap` relies on cross-platform native hooks.

## Phase 5: CI & Validation
*Status: Pending*
- Verified `cargo check --workspace` builds cleanly. The codebase is fully acyclic and desktop APIs are cross-platform compliant.
